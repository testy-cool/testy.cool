<Callout title="TL;DR" type="info">
UCP is an open standard (co-founded by Google and Shopify) that lets AI agents complete purchases via API. Instead of redirecting users to checkout pages, agents call standardized endpoints. This matters because AI agents currently can't buy things - every merchant speaks a different dialect. UCP creates a universal language.
</Callout>

---

## 1. What UCP Actually Is

UCP is a **specification** - not a product, not a platform, not a company. It defines how AI agents discover, negotiate with, and transact through any business that implements it.

```
// Without UCP
User clicks "Buy" ‚Üí Redirect to nike.com ‚Üí Agent loses context

// With UCP
User says "Buy" ‚Üí Agent calls UCP API ‚Üí Checkout happens programmatically
```

## 2. How It Works

### Discovery

Businesses publish their capabilities at a well-known URL:

```bash
curl https://nike.com/.well-known/ucp
```

If it returns JSON with capabilities, they support UCP. If 404, they don't.

### The Flow

<Steps>
  <Step>
    **Discovery** - Platform finds business's UCP profile
  </Step>
  <Step>
    **Negotiation** - Parties compute capability intersection (what both support)
  </Step>
  <Step>
    **Checkout** - Platform creates session, adds items, collects payment
  </Step>
  <Step>
    **Order Tracking** - Business sends webhooks for shipped, delivered, etc.
  </Step>
</Steps>

## 4. The Interesting Technical Bits

### Composable Capabilities

UCP doesn't force everyone to implement everything. Businesses publish what they support. Agents check compatibility. They transact using only the intersection.

```
// Business supports:
checkout + fulfillment + discounts

// Agent supports:
checkout + fulfillment

// They transact using:
checkout + fulfillment ‚úì
```

### Transport Agnostic

The spec is primarily REST, but also defines bindings for:
- **MCP** (Model Context Protocol)
- **A2A** (Agent-to-Agent)
- **Embedded** protocol for direct integration

Same capability, multiple transports.

### AP2 Mandates (Authorization Proof)

For autonomous agents making purchases, AP2 mandates provide cryptographic proof of authorization. When your agent makes a purchase, it includes a signed assertion that you actually authorized it.

<Callout type="warn">
**Why this matters:** Non-repudiation for high-value transactions. Proof that the right person authorized the charge.
</Callout>

### PCI-DSS Separation

- Platforms work with **opaque tokens**
- Businesses receive **tokenized instruments**
- Only certified PSPs touch **actual card numbers**

Raw credentials never flow where they shouldn't.

## 6. Does It Matter?

<Callout title="The Core Bet" type="info">
AI agents will increasingly handle purchasing decisions, and they need standard protocols to do so. If that happens, something like UCP becomes necessary. The alternative is the current mess - proprietary integrations, platform-specific plugins, manual checkout for anything outside walled gardens.
</Callout>

The timing question is whether agents are ready for autonomous purchasing. Right now, most people wouldn't trust an AI to buy things without confirmation. But the same was true of:
- Saving credit cards online
- Autofill
- One-click purchasing

The security design suggests the authors are thinking seriously about this. AP2 mandates, credential isolation, handler-based payment routing - these aren't afterthoughts.
