00:00 Overview

Introduction to Unsloth Studio

Unsloth is a powerful open-source framework that makes experimenting with and fine-tuning AI models incredibly efficient on consumer-grade hardware (like a GeForce RTX GPU).

They recently released Unsloth Studio, a visual, graphical interface. Instead of writing complex Python scripts, Studio lets you build data generation pipelines, fine-tune models, and interact with them in a chat interface directly from your browser.

00:34 Demo

The Goal: Teaching Models to "Think"

The objective of this tutorial is to take a small base model (which usually answers questions briefly and blindly) and teach it to "reason."

In the video demo, the models are asked: "What are the 3 primary colors?"

  • Base Model (Left): Instantly blurts out the answer with no explanation.
  • Fine-Tuned Model (Right): Takes a moment longer, outputting a <thinking> block explaining the color theory behind the answer, before delivering the final response.
00:52 Concept

How do we do this? Distillation

TL;DR: We use a massive, smart AI to write perfect, step-by-step answers. We then use those answers as a textbook to teach a smaller, faster AI.
🧠 Core Concept: Knowledge Distillation

Knowledge distillation involves two models: a Teacher and a Student. The Teacher is a massive model (like a 30B to 70B parameter model) that can reason deeply. The Student is a small model (like a 1B to 8B model) that runs fast. We prompt the Teacher to generate improved, detailed responses to hundreds of questions. We then use that generated dataset to train the Student model.

01:11 Setup

Prerequisites & Hardware Requirements

To follow along on your own machine, you will need the following:

  • Unsloth Studio: Installed via git and pip.
  • Hardware: A GPU with at least 16GB of VRAM (to fine-tune the small model). If you want to run the massive Teacher model locally with full GPU offloading, you'll need 24GB of VRAM.
  • Teacher Model: In this tutorial, we use Nemotron-3 Nano. It can be deployed locally or accessed via the cloud.

1. Install & Run Unsloth Studio:

$ git clone git@github.com:unslothai/unsloth
$ cd unsloth
$ pip install -e .
$ unsloth studio setup
$ unsloth studio -H 0.0.0.0 -p 8001

2. Deploy Teacher Model (Local llama.cpp):

$ ./unsloth/llama.cpp/build/bin/llama-server \
  -hf unsloth/Nemotron-3-Nano-30B-A3B-GGUF \
  --port 8000 \
  --alias my-local-nemotron \
  -n-gpu-layers all \
  --cache-size 0 \
  --ctx-size 2048
01:42 Architecture

The Synthetic Data Workflow

🧪 Core Concept: Synthetic Data Generation

Instead of humans writing thousands of examples of "how to think step-by-step," we use AI to generate the training data. This workflow is powered by NVIDIA NeMo Data Designer principles.

The workflow involves 3 distinct phases:

  • 1. Seed Dataset: We start with unsloth/alpaca-cleaned, a dataset containing basic instructions and generic outputs.
  • 2. Generation Prompts: We feed those instructions to our Teacher (Nemotron) and instruct it to rewrite the answers, adding a step-by-step reasoning phase.
  • 3. LoRA Fine-Tuning: We take the Augmented Training Dataset and use it to train the smaller Student model (OLMo-2 1B).
02:29 Pipeline Pt 1

Studio Recipes: Provider & Config Blocks

To build this pipeline, open Unsloth Studio, go to the Recipes tab -> New Recipe -> Start Empty.

1. Model Provider Block:
Tells Studio where your Teacher model lives. Provide an alias (e.g., "My Local Model Provider") and the endpoint: http://localhost:8000/v1. If using a remote provider (OpenRouter/Nvidia), enter your API key here.

2. Model Config Block:
Defines inference parameters for the Teacher. Set the alias to "Local Nemotron" and the exact model string: my-local-nemotron. Set Max Tokens to 2048.

Crucially, to enable Nemotron's built-in reasoning, add the following to the Optional Inference Body (JSON):

{
  "chat_template_kwargs": {
    "enable_thinking": true
  }
}
03:31 Pipeline Pt 2

Studio Recipes: Dataset & Prompts

Next, we structure the data flow:

3. Hugging Face Dataset Block:
Set the repository to unsloth/alpaca-cleaned. By default, this dataset has instruction, input, and output columns. We check the box to Drop the output column because we are going to generate a better one.

4. LLM Text Block:
This block talks to the Teacher. Name the output column reasoning_trace. Toggle "Drop from final dataset" (we'll format it first). Then, write the Prompt template.

🛠️ Tool: Jinja Templates

The prompt uses Jinja syntax (e.g. {{ instruction }}). For every row in the dataset, Studio dynamically pulls the text from the instruction column and injects it into this prompt.

5. Expression Block:
This final formatting block uses Jinja to wrap the generated text. We name the output column output_with_reasoning. The expression places the reasoning inside special tokens and appends the final answer:
<think>{{ reasoning_trace }}</think>{{ output }}

04:49 Generation

Executing Data Generation

With the pipeline complete, hit Validate at the bottom of the screen. Studio will verify the recipe logic.

Click Start Full Run. In the settings:

  • Give it a human-readable Run Name.
  • Set Records to 1000.
  • Enable Batching (makes the run more resilient to errors or timeouts).

The system will query your local Nemotron 1000 times. You can watch the terminal output to monitor progress. Once finished, you can click into the dataset to inspect the newly generated output_with_reasoning column to ensure it looks correct.

05:33 Training

QLoRA Fine-Tuning the Student

Navigate back to the main Studio tab to set up training.

⚡ Tool: QLoRA (Quantized LoRA)

Training a full model requires data-center supercomputers. QLoRA shrinks the base model down to 4-bit precision (saving memory) and only trains a tiny, specific layer of adapter weights on top of it. This enables training on a single standard GPU.

  • Model: Select allenai/OLMo-2-0425-1B-Instruct.
  • Method: QLoRA (4-bit).
  • Dataset: Select the Local dataset you just generated. Click the AI Assist button so Studio automatically maps Instruction -> User and output_with_reasoning -> Assistant.
  • Hyperparameters: Set Learning Rate to 0.0001. Open LoRA Settings and set Alpha to 32 (Rank 16). Set batch size to 10 and Max Steps to 100 (which equals exactly 1 epoch for our 1000 rows).

Click Start Training. A dashboard will appear tracking Training Loss, Gradient Norm, and Learning Rate. You want to see the Loss graph steadily decrease. This takes ~1 minute on a modern GPU.

07:11 Export

Testing & Exporting to GGUF

Once training is 100% complete, switch to the Chat tab. Under the "Fine-tuned" section, select your new model. Test it by asking a question. It should now successfully output a `` block before answering!

TL;DR: You can export your trained model as a GGUF file, allowing you to run it locally on almost any device (using CPU or GPU) via tools like LM Studio, Ollama, or llama.cpp.

To export:

  • Go to the Export tab.
  • Select your Training Run and the final Checkpoint.
  • Choose the GGUF / Llama.cpp Export Method.
  • Select a quantization level (Studio provides a recommended default).
  • Click Export Model.

Unsloth handles the complex conversion process in the background. Once finished, you have a highly-capable, custom-distilled reasoning model ready for local deployment!
